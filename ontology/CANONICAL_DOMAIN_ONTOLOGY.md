# CANONICAL_DOMAIN_ONTOLOGY.md

## Purpose

Prevent conceptual drift across integrated systems.

---

## Base Domains

truth  
artifact  
observation  
inference  
decision  
actor  
system  
policy  

---

## Domain Rules

- Every event must declare a domain.
- Cross-domain relations must be explicit.
- Domain expansion requires schema version update.
- No implicit domains allowed.

---

## Domain Integrity

Domain meaning must remain stable across versions.
Redefinition requires migration documentation.
